# OpenSpec Documentation

This directory contains all OpenSpec-related documentation and change management files for the Data Fusion Hub Service.

## Directory Structure

```
openspec/
├── project.md          # Project specification document
├── AGENTS.md           # Roles, responsibilities, and processes  
├── changes/            # Versioned change proposals and specifications
└── README.md           # This file
```

## Project Specification

The main project specification is located at `project.md` which contains:
- Overview and purpose of the Data Fusion Hub Service
- Tech stack and architecture patterns  
- Project structure and conventions
- Implementation status tracking
- Development environment setup instructions

## Change Management

All changes to the project are tracked in the `changes/` directory with timestamped filenames following this convention:
```
YYYYMMDD-HHMMSS-description.md
```

Each change file should contain a complete specification including:
- Title and summary  
- Detailed technical specifications
- Impact assessment
- Implementation plan

## Agents & Processes

The `AGENTS.md` file defines the roles, responsibilities, and processes for managing this project using OpenSpec principles.

This structure ensures consistent documentation practices and clear tracking of all project changes.
